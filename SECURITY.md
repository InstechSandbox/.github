# Security Policy

## Reporting A Vulnerability

If you believe you have found a security vulnerability in an InstechSandbox repository, do not report it through a public GitHub issue or public pull request.

Instead, report it privately to:

- `sandbox@instech.ie`

Include:

- the affected repository name
- a description of the issue
- steps to reproduce, if available
- impact assessment, if known
- any suggested mitigation, if you have one

## Expected Handling

InstechSandbox will review the report, assess severity, and decide the appropriate remediation path.

Because this organization includes proof-of-concept, protocol-facing, and deployment-related repositories, some issues may require coordinated fixes across more than one repository.

## Public Disclosure

Please allow a reasonable period for investigation and remediation before public disclosure.

If a repository needs stricter or more specific security reporting instructions, that repository should provide a repository-local `SECURITY.md` that overrides this default guidance.
